# 配置管理模块文档

## 1. 模块概述

配置管理模块用于管理SillyTavern的API连接配置和模型参数，支持多种API模式，包括文本补全、聊天补全、NovelAI和AI Horde。

## 2. 聊天补全模式表单

### 2.1 功能介绍

聊天补全模式表单用于配置聊天模式下的模型参数，支持以下功能：

- 基础参数配置：温度、频率惩罚、存在惩罚等
- 聊天相关参数配置：助手预填充、助手模拟等
- 提示词配置：主提示、模拟提示、越狱提示
- Prompts数组管理：动态增删、编辑prompts列表项

### 2.2 参数说明

#### 基础参数

| 中文名称 | 英文名称 | 类型 | 描述 | 功能与影响 | 默认值 |
|---------|---------|------|------|------------|-------|
| 温度 | temperature | 数字 | 控制token选择的随机性 | 值越高，输出越随机、创意性越强；值越低，输出越可预测、保守 | 1 |
| 频率惩罚 | frequency_penalty | 数字 | 惩罚频繁出现的token | 值越高，越抑制重复词汇的使用 | 0 |
| 存在惩罚 | presence_penalty | 数字 | 惩罚新token的使用 | 值越高，越倾向于使用已存在的token | 0 |
| 顶级P | top_p | 数字 | 控制token选择的多样性（核采样） | 值越高，考虑的token越多，输出越多样；值越低，只考虑高概率token，输出越集中 | 1 |
| 顶级K | top_k | 数字 | 限制每次选择的token数量 | 值越高，考虑的token越多，输出越多样；值越低，只考虑最可能的token，输出越集中 | 0 |
| 顶级A | top_a | 数字 | 通过绝对值切断低概率token | 值越高，过滤掉的低概率token越多 | 0 |
| 最小P | min_p | 数字 | 通过相对于最高token切断低概率token来限制token池 | 值越高，过滤掉的低概率token越多，输出越连贯但可能更重复 | 0 |
| 重复惩罚 | repetition_penalty | 数字 | 惩罚上下文中已出现的token | 值越高，越抑制重复；值越低，允许更多重复 | 1 |
| OpenAI最大上下文 | openai_max_context | 数字 | OpenAI最大上下文长度 | 值越大，模型可以处理的上下文越长，但可能会增加API调用成本 | 4095 |
| OpenAI最大Token数 | openai_max_tokens | 数字 | OpenAI最大生成Token数 | 值越大，生成的内容越长，但可能会增加API调用成本 | 300 |
| 名称行为 | names_behavior | 数字 | 控制名称在生成内容中的行为 | 不同的值会影响名称的处理方式 | 0 |
| 空发送内容 | send_if_empty | 字符串 | 当用户输入为空时发送的内容 | 设置用户空输入时的默认行为 | "" |

#### 聊天相关参数

| 中文名称 | 英文名称 | 类型 | 描述 | 功能与影响 | 默认值 |
|---------|---------|------|------|------------|-------|
| 助手预填充 | assistant_prefill | 字符串 | 助手回复的预填充内容 | 设置助手回复的初始内容 | "" |
| 助手模拟 | assistant_impersonation | 字符串 | 助手模拟用户的内容 | 设置助手如何模拟用户的风格和语气 | "" |
| 使用系统提示 | use_sysprompt | 布尔 | 是否使用系统提示 | 开启时使用系统提示，关闭时不使用 | false |
| 压缩系统消息 | squash_system_messages | 布尔 | 是否压缩系统消息 | 开启时压缩系统消息，减少上下文长度 | false |
| 媒体内联 | media_inlining | 布尔 | 是否启用媒体内联 | 开启时支持媒体内容内联显示 | true |
| 继续预填充 | continue_prefill | 布尔 | 是否启用继续预填充 | 开启时在继续生成时使用预填充内容 | false |
| 继续后缀 | continue_postfix | 字符串 | 继续生成时添加的后缀 | 设置继续生成时的后缀内容 | " " |
| 种子 | seed | 数字 | 设置随机种子 | 相同的种子会产生相同的输出，-1表示随机种子 | -1 |
| 生成数量 | n | 数字 | 设置生成的回复数量 | 值越大，生成的回复越多，但可能会增加API调用成本 | 1 |

#### 提示词配置

| 中文名称 | 英文名称 | 类型 | 描述 | 功能与影响 | 默认值 |
|---------|---------|------|------|------------|-------|
| 名称在补全中 | names_in_completion | 布尔 | 是否在补全中包含名称 | 开启时在生成内容中包含角色名称 | true |
| 主提示 | main_prompt | 字符串 | 设置主提示内容 | 控制模型的整体行为和角色定位 | "" |
| 模拟提示 | impersonation_prompt | 字符串 | 设置模拟提示内容 | 控制模型如何模拟用户的风格和语气 | "" |
| 越狱提示 | jailbreak_prompt | 字符串 | 设置越狱提示内容 | 控制模型突破限制的能力 | "" |

#### Prompts数组

Prompts数组是一个包含多个提示配置的列表，每个提示配置包含以下属性：

| 属性名 | 类型 | 描述 |
|-------|------|------|
| identifier | 字符串 | 提示标识符 |
| system_prompt | 布尔 | 是否为系统提示 |
| role | 字符串 | 角色 |
| name | 字符串 | 提示名称 |
| content | 字符串 | 提示内容 |

### 2.3 使用方法

1. 在API模式选择中选择"聊天补全"模式
2. 在模型参数页签中配置相关参数
3. 点击"保存参数"按钮保存配置
4. 点击"应用参数"按钮应用配置到SillyTavern

### 2.4 导入/导出功能

- **导入配置**：支持从JSON文件导入配置，自动处理不同格式的配置文件
- **导出配置**：支持将当前配置导出为JSON文件

## 3. 开发说明

### 3.1 代码结构

```
Config/
├── modes/
│   ├── AIHordeMode.tsx      # AI Horde模式配置
│   ├── ChatCompletionMode.tsx # 聊天补全模式配置
│   ├── NovelAIMode.tsx      # NovelAI模式配置
│   └── TextCompletionMode.tsx # 文本补全模式配置
├── ConfigManager.css        # 配置管理样式
├── ConfigManager.tsx        # 主配置管理组件
└── README.md                # 配置管理文档
```

### 3.2 核心功能

- **API模式切换**：支持文本补全、聊天补全、NovelAI、AI Horde四种模式
- **快速配置模板**：内置OpenAI、Anthropic、Mistral、TextGen、Ollama、LM Studio等模板
- **连接测试**：根据不同API模式发送测试请求，验证连接状态
- **模型参数设置**：根据不同API模式显示相应的参数配置界面
- **配置导入/导出**：支持从JSON文件导入配置，自动处理不同模式的配置格式

## 4. 注意事项

- 聊天补全模式下，模型参数会自动应用到SillyTavern
- 导入配置时，会自动处理字段映射和格式转换
- 保存配置时，会对数据进行验证，确保数据的正确性
- Prompts数组中的每个元素必须包含identifier、system_prompt、role、name和content五个属性

## 5. 故障排除

- **连接测试失败**：检查API地址是否正确，网络连接是否正常
- **配置保存失败**：检查配置名称是否为空，参数是否符合要求
- **导入配置失败**：检查导入的JSON文件格式是否正确
